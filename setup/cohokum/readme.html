Installation

Simply copy the file cohokum.exe into your /razorwroks/cohokum directory. You may want to rename your old cohokum.exe to save it. Note that if the eech.ini file does not exist it will be generated automatically. If you run into trouble it may be a good idea to delete the eech.ini file, possibly also the game.cfg file (your game settings).
If you have trouble configuring your joystick in eech.ini you can use a tool made by Retro (called eechmap.exe).
Check also the next paragraph for loadout screens.

There is a good thread at simhq, covering most of the installation issues up to 160w11: Installation and resource list . Some recent add-ons can be found at DevZone or one of the fan sites.

Retro has made a good little utility to make your joystick work with the current exe’s (you need to set the joystick axes in eech.ini). It can be downloaded from Kimchoc's site or here at simhq.

Complete installation

  1. Install Enemy Engaged Apache Havoc and run once to unpack graphics files
  2. patch EEAH up to 1.1E (patch can be found at Razorworks), run once
  3. Install Enemy Engaged Comanche Hokum
  4. patch to version 141c (patch can be found Razorworks)
  5. All newer versions are just replacement files for the executable cohokum.exe and can be copied into the "\razorworks\cohokum" directory, replacing the original:
    -     official versions 1.4.6X and 1.4.7X with improvements and compatibility with ATI radeon cards
    - unofficial version 151D which is 1.4.6/7X modified by DeStRo's Weapons and Units Tweaker to improve the game
    - unofficial versions 1.6.0W2 and later: modded versions using the official source code released by Razorworks
  6. The file game.cfg which contains the user settings (graphics, difficulty etc) may be changed when running a new version. In case of odd behaviour delete this file and re-configure the game to your liking.
  7. From version 1.6.0W5 onward cohokum .exe uses a configuration file eech.ini to control the game. This is a text file that can be altered by the user to confuigure the game. It is generated automatically if it doesn't exist. New mods and options are added automatically with each new version.
  8. For version 1.6.0W10 and later you need some extra files to fly blackhawk, hind etc: see here for more info
  9. Add-ons and more information can be found on the fan sites

NOTE: There's a very good short-cut in the installation procedure, after you've patched to 141c, d/l the "superpatchw14" from werewolfs site It holds all current add-on campaigns and skirmishes. At the same place you can d/l textures for the Alaska and Kuwait theatres. After d/l and extracting that to your RAzorworks folder (assuming you've used the default tree for an eech-installation), you can change to a newer exe-version by replacing cohokum.exe.

Flyable choppers

As of version W12 you can fly, change the loadout and repair the following choppers: Comanche, Apache Longbow, HokumB, Havoc, Blackhawk, Hind, AH64-A (Apache), and KA50 (Hokum). The cockpits and the flight behaviour of the new blue choppers default to the Apache Longbow, of the red choppers to the Havok. Make sure you download and install the additional download screens here: http://eech.bollocks.net.nz (in the addon download area). These zips have the full installation path , the files should go here: "\Razorworks\cohokum\graphics\ui\cohokum\Map\".

Note that you can pick a returning chopper to fly rogue. Spot the wanted Helo in the 2D-map, click on it, accept, auto-select (mission), confirm and off you go (you can't choose it from the mission or group list since it has no task).

Jabberwock made nay chopper flyable in free-flight mode. Just be sure you activate the “fly any aircraft” switch in eech.ini.
Then use F12-key and scroll through to spot the chopper you want to train on. Hit F12 again to clear the screen and “u” and you’re in the cockpit of the choosen chopper.

Files needed

What’s a WUT file?

A wut-profile (Weapons and Units Tweaker file) is a text file that alters some critical parameters in the game to make it more realistic/harder. The text file name is added to the file eech.ini that is read by cohokum.exe. The most used is the file 151_D.txt which basically increases speed and scan capabilities of aircraft and vehicles and power of various weapons. For more information you can visit http://home.planet.nl/~jette073/gotcha

If you intend to join a MP-game all three profiles are needed, you’re not aware of the hosts profile until you’re connecting and if the host is running with one of the 1_6_xxx-profiles your game will look for a similar one in your cohokum-folder. 1_6_13.txt is in all essential parts a replica of the old famous 151D-profile that was common before the source code was released. the file "1_6_13h.txt" is a bit harder version of 1_6_13.txt, adding aggressive enemy choppers. They will perform active scanning for you even if they’re on a recon or transfer mission and attack w/o provocation from your side :). The file " origwut.txt" is needed if you intend to play on-line and the host uses one of the other profiles. Origwut.txt restores the original settings when you leave a game with one of the other profiles.

NOTE: there is a new type of WUT file as of version 1.6.1W of cohokum but the old ones are also still operational.

What's NEW in EECH?

Text in red describes the additions to version 1.6.1 w

eech.ini parameter

Description

Look for master server

usemaster=0

When starting the game it reports to the master server and gets info from it (value = 1), when set to 0 the master server is ignored (default = 0)

GWUT and WUT files

wut=[filename]

WUT files are text files with Weapons and Units tweaks to make the game harder. As of version 160W15 a new version of the WUT file can be read as well as old style WUT files. For more information visit http://home.planet.nl/~jette073/gotcha

Server pause when client connects

pauseserv=1

When a host starts a game, he/she can hit "pause" and the game will resume when a client connects. When the (last) client leaves the game returns to "pause"-state and resumes again when a client connects (default=0, off).

Advanced Flight Model

advancedfm=0

If enabled, this setting switches to a more complicated flight model. Pedal mixing will be disabled, which means that you will have to use the pedals to turn the chopper. This makes it possible to fly sideways to track and attack moving vehicles, for example.(default=0, off) .

Session filtering

 

filter=0

You can filter the campaign screen to sort the campaign names according to their warzones.

Designated target list

 

destgt=0

Actvate target list (default =0). Make a list of targets to attack: Alt-NumEnter to add/remove targets to the list. You can select a special MFD filter (added to the low, medium, high that already exist), where you can select the targets with the regular keys. There are also two keypresses: Alt-Num0 and Ctrl-Num0, which allow to select designated targets even outside the filter (default =0, off).

Downwash effect

dwash=1

rotor downwash (dust) effect. It makes you see clouds of dust when the chopper gets close to the ground (default=1, on) . May give a framerate hit when activated.

Mouselook and TrackIR support

msl=1
msls=15

You can now control you view using the mouse or by using TrackIR. The TrackIR support uses enhanced mode which means that you will not have to bother about centering you view all the time. A check will be made for a TrackIR upon startup, and if it is found it will be used to control the view. Otherwise the mouse is used. (default = 0, off). Pressing the left mouse button will recenter the view to forward.
If mouselook is on, the msls setting controls the speed of the mouselook (6-20, default 15).

Mouse controlled FLIR

msl=0
msls=15

When mouselook is off (msl=0), you can use mouse for FLIR panning. Additionally, when you press left mouse button you lock/unlock target, with right button you can select next target and with mouse wheel you control zoom level.
It is also possible to switch mouselook on/off with ctrl-delete key combination or with a middle mouse button.
If mouselook is off, the msls setting controls the POV rotation speed (6-20, default 15).

Field of view

minfov=20
maxfov=80

The user can specify different field-of-view values, which can be activated by the '7','8' and '9' keys. 7 switches min FOV, 8 switches to (fixed) standard FOV, 8 switched to wide FOV. Please note that higher FOVs currently mean lower LODs.

Joystick controlled FLIR panning and zoom (Comanche and Hokum only)

eopann=-1
eopanv=6
eopanh=8

eozoomn=-1
eozoomax=7

You can now use joystick axis to pan the FLIR display. You specify the joystick number and which axis to use for this feature.

The old FLIR code only allowed three levels of zoom. This code has been updated to support any zoom levels. When using the keyboard, you still use the same keys to zoom, but you now have 5 levels instead. If you have an extra joystick axis available you can map one of them to the zoom and using for zooming.

DirectX numbers, general names, Saitek X36USB/X45 names:
1 - X-axis -  X-axis
2 - Y-axis -  Y-axis
3 - Z-axis
4 - rx-axis -  Rotary 2
5 - ry-axis
6 - rz-axis  - Rudder
7 - slider 1 - Throttle
8 - slider 2 - Rotary1

Mappable joystick axes

cyclicn=-1
cyclich=1
cyclicv=2

collectiven=-1
collectiveax=3

ruddern=-1
rudderax=6

You can now choose exactly which joystick and which axis on that joystick that should be used for cyclic, collective and throttle. See "FLIR panning" for DX numbers.

Ground stabilised FLIR (Comanche only)

grstab=1

The Comanche now supports ground stabilisation for the FLIR. You toggle this feature in-flight using Control-S.

Fly any aircraft

faa=1

Fly any aircraft: all helicopters available now for the user. Fly your own insertions! The cockpit defaults to the Apache when choosing anything else than CHAH (default = on) .

Wideview cockpit

commanche pilot=x,y,z
commanche co-pilot=x,y,z
hokum pilot=x,y,z
hokum co-pilot=x,y,z
apache pilot=x,y,z
havoc pilot=x,y,z

Retreating the pilots head causes the player to have a wider field of view without distortion. The values can be edited by pressing Ctrl-\ in the game and using the numpad keys to move the viewpoint: 4 and 6 move distance to dashboard; 2 and 8 move vertically; 1 and 3 move horizontally. Alt-\ quits editing and saves the values in the eech.ini file. (Note: the file widemod.cfg is no longer needed).

Note: for some keyboards the keys are not Ctrl-\ and Alt-\ but something else. You just have to try for yourself on the right hand side of the keyboard. For more info: http://home.planet.nl/~jette073/gotcha

Show framerate

dfr=0

The framerate is displayed in the top left corner and updates about once a second. The average of 30 readings is shown for stability. Values: 0 = off; 1 = on; 2 write the framerate to a file framerate.txt in the cohokum directory .

Multiplayer master server

 

At http://hoxdna.org/eech there is a masterserver for EECH, listing all running games along with total players and IP addresses. EECH versions newer than 1.6.0W1 report to the masterserver if you play a game with Networking set to TCP/IP in the options menu. To join a game, just copy&paste an active IP from the serverlist into GSS (Ground Support System) or EECH itself.

High resolution MFD

highresmfd=0

The resolution of the Comanche and Hokum MFD's will be doubled. Remember that there is a small framerate hit when activating this feature.

TSD render mod

tsdrender=1
tsdpalette=2
tsdenemy=0

This mod adds a relief map background to the TSD of the Comanche, Hokum and Apache (the Havoc is not included yet). The map is shown as elevation colours in several colour palettes: green/yellow/brownish colours and water is blue. The maximum elevation of the map is used to scale the colours. Shaded relief is also added as an option.

  • The Apache now has the same TSD as the Comanche.
  • Optional: the friendlies and enemies are shown in red and blue colours (including detection circles of SAMs). If you don't want this option set the eech.ini variable "tsdenemy" to 0.
  • The configuration is automatically saved in the eech.ini file when you leave the game.
  • There is a small framerate hit when showing the terrain.

ressing Ctrl-D (as in Declutter) cycles through the options: old TSD screen, the terrain map with/without contours, shaded relief with/without contours. Alt-D cycles through the colour palettes (0-2). For more info: http://home.planet.nl/~jette073/gotcha.

Radar ranges

radar_comanche = a,b,c,d,e
radar_apache = a,b,c,d,e
radar_hokum = a,b,c,d,e radar_havoc = a,b,c,d
radar_blackhawk = a,b,c,d,e
radar_hind = a,b,c,d

Adjustable radar ranges: a,b,c,d,e are the ranges, default are for instance: 500, 1000, 2000, 4000, 8000 for the Comanche. Note that the Havoc and Hind have only 4 ranges. If you consider this a cheat use the default ranges.

NOTE: THIS IS NO LONGER NEEDED AND IGNORED IF YOU USE THE NEW GWUT FILE

Amount of cannon ammo

rounds_comanche = x
rounds_apache = x
rounds_hokum_AP = x
rounds_hokum_HE = x
rounds_havoc_AP = x
rounds_havoc_HE = x
rounds_blackhawk = x
rounds_hind_AP = x
rounds_hind_HE = x

500 rounds of 20 mm M197 ammo? Common... be serious! Put in an amount that seems reasonable to you (max 65000). You can give each chopper its own amount. Leave the values at default if you consider this a cheat .

NOTE: THIS IS NO LONGER NEEDED AND IGNORED IF YOU USE THE NEW GWUT FILE

Satellite view

 

Ctrl-F9 in game gives you another camera view: satellite, both for your own gunship and for other groups. Moreover, pressing Space on the map view when a site is selected switches the view to satellite camera. When a group is selected you can zoom satellite view with default controls, for keysites it is also possible to pan view. Note that at the moment for debugging purposes the level of interference is selected randomly each time the satellite view is select ed. In the future it will be linked to weather/daytime.

Mouse map control

 

The map view may be zoomed in/out with a mouse wheel. It is also possible to pan the map when the right button is pressed (it needs some improvement still). When the small 3d preview window is selected, mouse movements and wheel with ctrl pressed will pan/zoom the camera.

Campaign commander

camcom=1

When the camcom setting is set to 1 (def = 0), it is possible to designate targets on the main campaign map. Just select the enemy target and press left mouse button: a new task is created. New tasks work the same as other tasks: they may not be executed right away, may expire etc. When you press right mouse button over a task, it is immediately terminated. If it was assigned to any group, the group is sent home.
Note that Campaign Commander greatly influences the gameplay balance: with the right planning the campaigns are much easier to win.

Enhanced inset view

 

Now you can select inset view (Alt-F12) also in the cockpit. 0 key selects target (default). The keys 2-6 select the same views as corresponding padlock commands.

Note: two bugs have been more or less solved: the "FARP" bug that caused helicopters to cue at landing pads, and the "double mouse" bug.

A new bug appeared with the test-versions after W12. It seems to origin from erroneous entries in the WUT-files. Therefore new WUT-files are packed together with this exe. All should be put in the cohokum-folder. origwut.txt is for restoring the default game-profile after joining a game with tweaked profile. 1_6_13.txt is the essential parts of the old 151D-profile except for the weapons part. Weapons can now be tweaked in the eech.ini-file. 1_6_13H.txt is an even harder version of the 151D-pro file –enemy choppers will actively search for you and attack.
