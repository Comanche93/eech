What's NEW in EECH 161W4?

  1. A fix for the new Saitek X52 controller (your mouse button cannot be defined as axes and the limit is 32 buttons but it works).
  2. With this version you can use the Apache Havoc maps (Georgia, Thailand, Cuba) without having installed Enemy Engaged Apache Havoc (we have permission from Razorworks, thanks Todd Gibbs for intervening). This feature is automatic. The maps can be downloaded from http://hoxdna.org/eech/files/ (thanks werewolf). Note that the setup file is ~120 Mbyte !
  3. The Cannon may follow the viewpoint without autolocking the targets. This works when acqusition systems are off or when IHADSS/HIDSS/HMS is used - depending on the eech.ini setting. Works with TrackIR! You switch in-game with Ctrl-NumDel.
  4. Custom made textures: textures can now be read from a texture directory and its subdirectories, instead of having to import them in the texture.bin or texture.pal files. Campaigns can have warzone specific textures in special directories. You can add directories with your own textures if you want. You need the file texturesetup.exe from Gotcha's site. (Note: no alpha blending in textures yet)
  5. The UI control page has been extended with a lot of features: automatic joystick recognition, mouselook, TrackIR etc.
  6. Joystick deadzone fix and new mouselook handling.
  7. A first attempt at a 3D Apache cockpit in-game. It uses only in game models and its ugly but its a start. Set 3d_cockpit=1 in eech.ini.
  8. If you don't like the heavy black bars in front of your face in the old Apache cockpit, you can switch to wideview and make it partly disappear by pressing: first Ctrl-\, then num-5 to set X=-8 and num-1 and num-3 to fine tune. The bar starts to disappear if X reaches a value of about -8.200.
  9. A high LOD hack, more detail, less fps. Set high_lod_hack=1 in eech.ini.
  10. The radar ranges and ammo tweaks have disappeared from the eech.ini file. Use the GWUT editor to add those to your WUT file.
  11. Some smaller bug fixes

Installation

  • If you used eechsetup161Wx.exe installed then you are set. If you go online you will need to edit eech.ini manually and probably add a WUT filename if the host wants it.
  • If you experience problems, delete eech.ini and game.cfg and let the game generate them again. You will have to reset the ingame settings (graphics, sound, difficulty etc).
  • If you didn't use the installer, simply copy the file cohokum.exe into your /razorworks/cohokum directory. You may want to rename your old cohokum.exe to save it.
  • If you have joystick problems, Retro has made a good little utility to make your joystick work with the current exe’s (you need to set the joystick axes in eech.ini). It can be downloaded from Kimchoc's site or here at simhq. Note that the control page in the new versions has been improved!

Complete installation for EEAH and EECH

There is a good thread at simhq, covering most of the installation issues up to 161w: Installation and resource list . Some recent add-ons can be found at DevZone or one of the fan sites.

  1. Install Enemy Engaged Apache Havoc and run once to unpack graphics files
  2. Patch EEAH up to 1.1E (patch can be found at Razorworks), run once
  3. Install Enemy Engaged Comanche Hokum
  4. Patch to version 1.4.1C (patch can be found Razorworks)
  5. All versions newer than 1.4.1C are just replacement files for the executable cohokum.exe and can be copied into the "\razorworks\cohokum" directory, replacing the original. This is true for the offcial replcaments 1.4.6X and 1.4.7X, and for all the modded ones created since the source code was released. The moded exes are numbered 1.6.0Wxx until 1.6.1Wxx.
  6. The file game.cfg which contains the user settings (graphics, difficulty etc) may be changed when running a new version. In case of odd behaviour delete this file and re-configure the game to your liking.
  7. From version 1.6.0W5 onward cohokum .exe uses a configuration file eech.ini to control the game. This is a text file that can be altered by the user to confuigure the game. It is generated automatically if it doesn't exist. New mods and options are added automatically with each new version.
  8. Add-ons and more information can be found on the fan sites.

Game info

Flyable choppers

As of version 1.6.0.W12 you can fly, change the loadout and repair the following choppers: Comanche, Apache Longbow, HokumB, Havoc, Blackhawk, Hind, AH64-A (Apache), and KA50 (Hokum). The cockpits and the flight behaviour of the new blue choppers default to the Apache Longbow, of the red choppers to the Havok.

If you didn't use the eechsetup161W.exe make sure you download and install the additional download screens here: http://eech.bollocks.net.nz (in the addon download area). These zips have the full installation path, the files should go here: "\Razorworks\cohokum\graphics\ui\cohokum\Map\".

Note that you can pick a returning chopper to fly rogue. Spot the wanted Helo in the 2D-map, click on it, accept, auto-select (mission), confirm and off you go (you can't choose it from the mission or group list since it has no task). You can only do this if the option "faa" in eech.ini is set to "1" (see below).

Jabberwock made any chopper flyable in free-flight mode. Just be sure you activate the “fly any aircraft” switch in eech.ini. Then use F12-key and scroll through to spot the chopper you want to train on. Hit F12 again to clear the screen and “u” and you’re in the cockpit of the choosen chopper.

Multiplayer status

Jabberwock, with the support from Todd Gibbs, has made some changes to the server-part of the code. Tested out OK by Moje, but the test environment is limited, so there could still be bugs left. It's crucial that the host runs with cvc=1.

What’s a WUT file?

Most people fly with a WUT file. A WUT-profile (Weapons and Units Tweaker file) is a text file that alters some critical parameters in the game to make it more "realistic" and harder. The text file name can be added to the file eech.ini and all parameters are read by cohokum.exe at startup. The most used WUT file is gwut151D.csv which basically increases speed and scan capabilities of aircraft and vehicles and power of various weapons (increasing gameplay difficulty). Furthermore there is gwut1613H.csv which adds enemy chopper aggressiveness (all enemy choppers behave as if they are on a strike mission), and gwut146X.csv which is needed when playing multiplayer and has the game's default values (identical to playing without a WUT file).

You can edit these files with the GWUT editor, downloadable from Gotcha's site. If you intend to join a MP-game all three WUT files are needed. You are not aware of the host's profile until you’re connecting and if the host is running with one of the WUT profiles your game will look for a similar one in your cohokum-folder.

NOTE: there are also older type WUT files that have less information and are called 151_d.txt, 1_6_13h.txt and origwut.txt. These can still be used and have the same behaviour but are no longer supported.

The eech.ini file and mod info

In order of appearance in eech.ini:

[Communications]

Look for master server

usemaster=0

When starting the game it reports to the master server and gets info from it (value = 1), when set to 0 the master server is ignored (default = 0)

Multiplayer master server

pss=[server name]

sss=[server name]

Primary and secondary server setting. At http://hoxdna.org/eech there is a master server for EECH, listing all running games along with total players and IP addresses. EECH versions newer than 1.6.0W1 report to the master server if you play a game with Networking set to TCP/IP in the options menu. To join a game, just copy&paste an active IP from the server list into GSS (Ground Support System) or EECH itself.

Server pause when client connects

pauseserv=1

When a host starts a game, he/she can hit "pause" and the game will resume when a client connects. When the (last) client leaves the game returns to "pause"-state and resumes again when a client connects (default=0, off).

[Dynamics]

Advanced Flight Model

advancedfm=0

If enabled, this setting switches to a more complicated flight model. Pedal mixing will be disabled, which means that you will have to use the pedals to turn the chopper. This makes it possible to fly sideways to track and attack moving vehicles, for example.(default=0, off) .

[Radar ranges (obsolete)]
 

Adjustable radar ranges: this feature is available through the GWUT files

[Amount of cannon ammo (obsolete)]
 

Adjustable ammo: this feature is available through the GWUT files

[WUT]

wut=[filename]

WUT files are text files with Weapons and Units Tweaks to make the game harder. As of version 160W15 the new version "GWUT" files can be read as well as old style WUT files. For more information visit Gotcha's site.

[wideview]

Change your point of view to see more of the cockpit

commanche pilot=x,y,z
commanche co-pilot=x,y,z
hokum pilot=x,y,z
hokum co-pilot=x,y,z
apache pilot=x,y,z
havoc pilot=x,y,z

Retreating the pilots head causes the player to have a wider field of view without distortion. The values can be edited by pressing Ctrl-\ in the game and using the numpad keys to move the viewpoint: 4 and 6 move distance to dashboard; 2 and 8 move vertically; 1 and 3 move horizontally. Alt-\ quits editing and saves the values in the eech.ini file.

Note: for some keyboards the keys are not Ctrl-\ and Alt-\ but something else. You just have to try for yourself on the right hand side of the keyboard. For more info: Gotcha's site

[Mods]

Mouselook and TrackIR support

msl=1
msls=15

You can now control you view using the mouse or by using TrackIR. The TrackIR support uses enhanced mode which means that you will not have to bother about centering you view all the time. A check will be made for a TrackIR upon startup, and if it is found it will be used to control the view. Otherwise the mouse is used. (default = 0, off). Pressing the left mouse button will recenter the view to forward.
If mouselook is on, the msls setting controls the speed of the mouselook (6-20, default 15).

Mouse controlled FLIR

msl=0
msls=15

When mouselook is off (msl=0), you can use mouse for FLIR panning. Additionally, when you press left mouse button you lock/unlock target, with right button you can select next target and with mouse wheel you control zoom level.
It is also possible to switch mouselook on/off with ctrl-delete key combination or with a middle mouse button.
If mouselook is off, the msls setting controls the POV rotation speed (6-20, default 15).

Field of view

minfov=20
maxfov=80

The user can specify different field-of-view values, which can be activated by the '7','8' and '9' keys. 7 switches min FOV, 8 switches to (fixed) standard FOV, 9 switched to wide FOV. Please note that higher FOVs currently mean lower LODs.

Joystick controlled FLIR panning and zoom (Comanche and Hokum only)

eopann=-1
eopanv=6
eopanh=8

eozoomn=-1
eozoomax=7

You can now use joystick axis to pan the FLIR display. You specify the joystick number and which axis to use for this feature.
The old FLIR code only allowed three levels of zoom. This code has been updated to support any zoom levels. When using the keyboard, you still use the same keys to zoom, but you now have 5 levels instead. If you have an extra joystick axis available you can map one of them to the zoom and using for zooming.
DirectX numbers, general names, Saitek X36USB/X45 names:
1 - X-axis -  X-axis
2 - Y-axis -  Y-axis
3 - Z-axis
4 - rx-axis -  Rotary 2
5 - ry-axis
6 - rz-axis  - Rudder
7 - slider 1 - Throttle
8 - slider 2 - Rotary1

Mappable joystick axes

cyclicn=-1
cyclich=1
cyclicv=2

collectiven=-1
collectiveax=3

ruddern=-1
rudderax=6

You can now choose exactly which joystick and which axis on that joystick that should be used for cyclic, collective and throttle. See "FLIR panning" for DX numbers.

Show infantry on radar

radarinf=1

infantry no longer visible on radar, def = 1 (on)

Ground stabilised FLIR (Comanche only)

grstab=1

The Comanche now supports ground stabilisation for the FLIR. You toggle this feature in-flight using Ctrl-S.

Show framerate

dfr=0

The framerate is displayed in the top left corner and updates about once a second. The average of 30 readings is shown for stability. Values: 0 = off; 1 = on; 2 write the framerate to a file framerate.txt in the cohokum directory. This is not a very good measure, you better use the FRAPS tool.

Downwash effect

dwash=1

rotor downwash (dust) effect. It makes you see clouds of dust when the chopper gets close to the ground (default=1, on) . May give a framerate hit when activated.

High resolution MFD

highresmfd=0

The resolution of the Comanche and Hokum MFD's will be doubled. Remember that there is a small framerate hit when activating this feature.

Green MFD

greenmfd=1

MFD's are green (1) or grey (0).

Fly any aircraft

faa=1

Fly any aircraft: all helicopters available now for the user. Fly your own insertions! The cockpit defaults to the Apache when choosing anything else than CHAH (default = on) .

Special TSD render

tsdrender=1
tsdpalette=2
tsdenemy=0

This mod adds a color and/or shaded relief map background to the TSD of the Comanche, Hokum and Apache (the Havoc is not included). Ctrl-D cycles through the options. There are 3 palettes to choose from for the elevation (Alt-D). The Apache has the same TSD as the Comanche. The configuration is automatically saved in the eech.ini file when you leave the game.
Optional: the friendlies and enemies are shown in red and blue colours (including detection circles of SAMs). If you don't want this option set the eech.ini variable "tsdenemy" to 0. There is may be a framerate hit when showing the terrain. For more info: Gotcha's site .

Campaign commander

camcom=0

When the camcom setting is set to 1 (def = 0), it is possible to designate targets on the main campaign map. Just select the enemy target and press left mouse button: a new task is created. New tasks work the same as other tasks: they may not be executed right away, may expire etc. When you press right mouse button over a task, it is immediately terminated. If it was assigned to any group, the group is sent home. Note that Campaign Commander greatly influences the gameplay balance: with the right planning the campaigns are much easier to win.

Designated target list

destgt=0

Actvate target list (default =0). Make a list of targets to attack: Alt-NumEnter to add/remove targets to the list. You can select a special MFD filter (added to the low, medium, high that already exist), where you can select the targets with the regular keys. There are also two keypresses: Alt-Num0 and Ctrl-Num0, which allow to select designated targets even outside the filter (default =0, off).

Session filtering

filter=0

You can filter the campaign screen to sort the campaign names according to their warzones.

Pedal input

reverse_pedal=0

reversed pedal input

TrackIR external view

external_trackir=0 if TrackIR is active, let it control external view too
High LOD
high_lod_hack=0 EXPERIMENTAL! Enables highest level-of-detail models at far distances. Nice for higher FOVs, bad for FPS (esp. near cities). Default = 0
3D apache cockpit
3d_cockpit=0 EXPERIMENTAL! Draws a 3D apache cockpit using an in game apache model. Very ugly but fun.
Default = 0;
Cannon tracking boresight
cannontrack=1 Cannon is tracking the pilot's view. Three settings are available: 0 = tracking off, 1 = tracking only when no acquisition system is turned off (Ctrl-NumDel), 2 = tracking only when IHADSS/HIDSS/HMS system is active.

Extra In-game commands

Designated target list
Alt-Num0
Ctrl-Num0

Allows you to select designated targets even outside the target filter (see designated target list mod above)

Enhanced inset view

Alt-F12

Inset view in the cockpit. Pressing 0 selects the target (default). Pressing keys 2-6 select the same views as corresponding padlock commands.

Field of view

7,8,9

Activates near, normal, wide field of view (FOV)

Ground stabilised FLIR (Comanche only)

Ctrl-S

Toggles ground stabilized FLIR on/off.

Mouse controlled view panning

Middle mouse
/ Ctrl-Del

Switches between mouselook for panning and fixed view (whre F1 and F2 work). Fixed view means that the mouse can be used for the FLIR.

Mouse controlled FLIR

Left-mouse
Right-mouse
Middle mouse/Ctrl-Del

When mouselook is off (msl=0), you can use mouse for FLIR panning. Additionally, when you press left mouse button you lock/unlock target, with right button you can select next target and with mouse wheel you control zoom level. It is also possible to switch mouselook on/off with ctrl-delete key combination or with a middle mouse button.

Cannon tracking
Ctrl-Num Del
 Cannon is tracking the pilot's view. Three settings are available: 0 = tracking off, 1 = tracking only when no acquisition system is turned off (Ctrl-NumDel), 2 = tracking only when IHADSS/HIDSS/HMS system is active.
Mouse map control

Mouse wheel

The map view may be zoomed in/out with a mouse wheel. It is also possible to pan the map when the right button is pressed (it needs some improvement still). When the small 3d preview window is selected, mouse movements and wheel with ctrl pressed will pan/zoom the camera.

Satellite view

Ctrl-F9

Pressing Ctrl-F9 gives you satellite view, both for your own gunship and for other groups. Moreover, pressing Space on the map view when a site is selected switches the view to satellite camera. When a group is selected you can zoom satellite view with default controls, for keysites it is also possible to pan view. Note that at the moment for debugging purposes the level of interference is selected randomly each time the satellite view is select ed. In the future it will be linked to weather/daytime.
Special TSD render

Ctrl-D
Alt-D

Ctrl-D (as in Declutter) cycles through the TSD render options: old TSD screen, the terrain map with/without contours, shaded relief with/without contours. Alt-D cycles through the elevation colour palettes (0-2).
Wideview editing

\
Crtl-\
Alt-\
numpad keys 1-9

Switch between normal view and wideview by pressing "\", edit the view position by pressing "Ctrl-\" and then the numpad keys and save the new position with "Alt-\". The values are stored in eech.ini. Note that on non-English keyboards a different key may be needed (in stead of "\"). Numpad keys can be used to shift the viewpoint: num-8 and num-2 for the vertical, num-4 and num-6 for the distance.

Note: when you fly the Apache and you don't like the heavy black bars in front of your face, you can switch to wideview and make it partly disappear by pressing: first Ctrl-\, then num-5 to set X=-8 and num-1 and num-3 to fine tune. The bar starts to disappear if X reaches a value of about -8.200.
Campaign commander
Left and right Mouse
 Just select the enemy target and press left mouse button: a new task is created. When you press right mouse button over a task, it is immediately terminated. Works when option camcom=1 in the eech.ini file.
